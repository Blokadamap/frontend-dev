# Блокадная карта

## Стек

- **React 19** – библиотека для интерфейсов
- **TypeScript** – типизация
- **Vite** – сборка и дев-сервер
- **React Router v7** – навигация
- **TanStack Query (v5)** – работа с API (запросы, кэширование)
- **Jotai** – атомарное состояние (фильтры, выбранный слой)
- **Axios** – HTTP-клиент
- **React Leaflet** – карты (обёртка над Leaflet)
- **Lucide React** – иконки

## Структура проекта

- `public/` – статические файлы, которые копируются без изменений (favicon, robots.txt)
- `src/`
  - `assets/` – изображения, шрифты (импортируются в компоненты)
  - `components/` – переиспользуемые UI-компоненты (Button, Modal, Input)
  - `pages/` – страницы приложения
    - `Landing/` – лендинг (компоненты и стили)
    - `Map/` – страница с картой (компоненты, хуки, стор)
  - `hooks/` – общие кастомные хуки (useDebounce, useLocalStorage)
  - `lib/` – настройки библиотек (axios, queryClient)
  - `services/` – функции для работы с API
  - `types/` – глобальные TypeScript-типы
  - `utils/` – вспомогательные функции
  - `App.tsx` – роутинг
  - `main.tsx` – точка входа

## Скрипты

| Команда | Описание |
|---------|----------|
| `dev`   | Запускает сервер для разработки с горячей перезагрузкой (HMR). Адрес: `http://localhost:5173` |
| `build` | Собирает оптимизированную версию приложения для продакшена в папку `dist/`. Перед сборкой проверяет типы TypeScript (`tsc -b`). |
| `lint`  | Запускает ESLint для проверки кода на ошибки и соответствие стилю. |
| `preview` | Локально предпросматривает собранный проект (предварительно выполните `build`). Адрес: `http://localhost:4173` |

> **Примечание:** перед первым запуском выполните `yarn install`.

## Работа с ветками (Git Flow)

Проект использует упрощённый Git Flow:

- **`main`** – основная ветка. Здесь всегда стабильная версия, готовая к деплою.
- **`feature/*`**, **`fix/*`**, **`refactor/*`** – ветки для разработки новых фич, исправлений и рефакторинга.

### Когда использовать feature, fix, refactor?

Чтобы в названии ветки сразу было понятно, что именно делается, используем префиксы:

| Префикс       | Когда применять                                                                 | Примеры                        |
|---------------|---------------------------------------------------------------------------------|--------------------------------|
| **feature/**  | Добавление новой функциональности: новая страница, компонент, кнопка, фильтр и т.п. | `feature/map-layers` <br> `feature/filter-panel` |
| **fix/**      | Исправление ошибок (багов): что-то работало неправильно и ты это чинишь.          | `fix/popup-closing` <br> `fix/map-center` |
| **refactor/** | Улучшение кода без изменения поведения: переименование, разбивка на компоненты, удаление дублирования. | `refactor/rename-components` <br> `refactor/extract-hook` |

**Главное правило:** название ветки должно быть кратким, понятным и отражать суть изменений. Если сомневаешься, какой префикс выбрать – используйте `feature/*`
